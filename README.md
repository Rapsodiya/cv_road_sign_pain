# cv_road_sign


# Семантическая сегментация дорожных знаков

Курсовая работа по дисциплине «Нейронные сети / Компьютерное зрение».

---

## Описание задачи

Задача — **попиксельная семантическая сегментация дорожных знаков** на фотографиях. Модель должна для каждого пикселя изображения определить: принадлежит ли он знаку или фону. Реализованы и сравниваются две архитектуры:

- **YOLOv11n-seg** — современная one-stage модель с instance segmentation, предобученная на COCO.
- **U-Net (ResNet34 encoder)** — классическая encoder-decoder архитектура со skip-connections.

---

## Датасет

**Russian Road Signs Segmentation Dataset**
Источник: [Kaggle](https://www.kaggle.com/datasets/viacheslavshalamov/russian-road-signs-segmentation-dataset)

Аннотации выполнены в формате **VIA (VGG Image Annotator)** — полигональные маски, описывающие точный контур каждого знака.


## Архитектура U-Net (UNetResNet34)

В работе реализована U-Net с энкодером на базе **ResNet34**, предобученного на ImageNet.

### Принцип работы

U-Net состоит из трёх частей:

**Encoder (сжатие)** — последовательно уменьшает пространственное разрешение и увеличивает число каналов, извлекая всё более абстрактные признаки. 

**Bottleneck (дно)** — самый глубокий уровень, DoubleConv(256 → 512). Здесь сжатое представление изображения.

**Decoder (восстановление)** — зеркально расширяет карты признаков обратно до исходного разрешения. На каждом уровне декодера используется **skip connection** — конкатенация с картой признаков соответствующего уровня энкодера. Это позволяет восстанавливать мелкие детали, потерянные при сжатии.


### Почему ResNet34 в роли энкодера?

ResNet34 предобучена на ImageNet — это значит, что энкодер уже умеет извлекать низкоуровневые признаки (края, текстуры) и высокоуровневые (формы, объекты). При малом датасете это критически важно: обучение с нуля потребовало бы в несколько раз больше данных для получения сопоставимого качества.

## Функция потерь

Для U-Net использована **комбинированная функция потерь**: BCE + Dice Loss.

**BCE (Binary Cross-Entropy)** — стандартная функция для бинарной классификации каждого пикселя:

**Dice Loss** — напрямую оптимизирует метрику IoU (пересечение / объединение):

Dice Loss штрафует именно за отсутствие пересечения с разметкой, что заставляет модель находить объекты даже при сильном дисбалансе.

## Метрики качества

### IoU (Intersection over Union / Jaccard Index)

### Precision (Точность)

Доля пикселей, которые модель пометила как знак и действительно оказались знаком. Низкая Precision — много ложных срабатываний (модель «рисует» маску там, где знака нет).


### Recall (Полнота)

Доля пикселей реального знака, которую модель нашла. Низкий Recall — модель пропускает знаки (false negatives).

**mAP (mean Average Precision)** — стандартная метрика для детекции объектов. Считается как площадь под кривой Precision-Recall при заданном пороге IoU для совпадения предсказания с ground truth.

- `mAP@0.5` — порог IoU = 0.5 (предсказание засчитывается, если IoU ≥ 0.5)
- `mAP@0.5:0.95` — усреднение по порогам от 0.5 до 0.95 с шагом 0.05 (стандарт COCO, строже)


### L2 (Euclidean distance между центроидами, только YOLO)

Евклидово расстояние между центром предсказанной маски и центром ground truth маски в пикселях. Показывает, насколько точно модель локализует знак, даже если форма маски неточная.

## Аугментации

Для противодействия переобучению при малом датасете применяются:

- **ColorJitter** — случайные изменения яркости, контраста, насыщенности
- **GaussNoise** — добавление шума (имитация камер низкого качества)
- **RandomRotate90** — случайные повороты на 90°
- **HorizontalFlip** — горизонтальное отражение
- **Нормализация ImageNet**

## Выводы

- **YOLOv11-seg** показывает более высокую скорость инференса и лучше справляется с несколькими объектами на изображении.
- **U-Net с ResNet34** за счёт предобученного энкодера достигает высокого IoU даже с ограниченным датасетом.
- Главная сложность — малый размер объектов (< 5% площади). BCE + Dice Loss и аугментации существенно помогают с этой проблемой.
- Карты ошибок (FP/FN) позволяют наглядно увидеть, где именно модель ошибается.
